evolution simulator

The purpose of this program is to simulate the basic evolution of virtual creatures via natural selection. Creatures and plants are represented in data by pointers to C++ classes that are stored in one dimensional arrays. Upon the start of the program, 44 creatures (40 herbivores and 4 carnivores) are instantiated at random locations with all traits set to 128. A task queue is then initialized to assist with the program’s execution. In an infinite loop, the actions of the creatures for this tick are determined. This step is run in parallel via a thread queue as all creatures can act independently.

Afterwards, all creatures have their positions updated and collisions are detected and handled. Collisions require serial execution as this step requires creatures to be handled and modified in pairs and therefore doesn’t lend itself well to parallelization. To compensate, great care was taken to ensure only the code strictly necessary for a creature’s processing is run. After all creatures have been updated, new plants are generated. Plants and creatures are then drawn on a bitmap and statistics concerning the state of the simulation are recorded. The updated bitmap is displayed, and the program sleeps until a predetermined amount of time has passed between the start and end of this tick to preserve visual continuity.

This program was created as a final project for CSC 216: Operating Systems and Parallel Algorithms at Grinnell College, in the spring of 2016. See the code and run it for yourself here.
